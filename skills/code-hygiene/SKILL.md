## 概述

AI 生成的代码有特定的"异味"模式——过度防御、废话注释、不必要的复杂度。这些模式降低代码可读性和可维护性。

> 本 skill 改编自 [OmO](https://github.com/code-yeongyu/oh-my-openagent) 的 `ai-slop-remover` skill，调整为预防性约束（在编写时避免）而非事后清理。

## 规则

### 1. 注释卫生

**禁止的注释模式：**

- 复述代码的注释：`x += 1  // x 加 1`
- 显而易见的文档注释：`/** 返回名称 */ getName()`
- 分隔线注释：`// ===== 辅助函数 =====`
- 无具体计划的 TODO：`// TODO: 未来优化`
- 无解释的"重要"标记：`// 注意：这很重要`

**允许的注释：**

- 解释"为什么"的注释（业务逻辑、边界条件、变通方案）
- 引用 issue/ticket 的注释：`// 见 #1234`
- 非显而易见的算法解释
- 正则表达式解释
- 与项目现有风格一致的注释

**判断标准：** 如果删掉注释后，一个熟悉该语言的开发者仍能理解代码，那这个注释就是多余的。

### 2. 防御性代码卫生

**禁止的过度防御模式：**

- 对不可能为 null 的值做 null 检查
- 对静态类型参数做 `instanceof` 检查
- 对不会抛异常的代码加 try-catch
- 给必填参数设置无意义的默认值
- 已删除代码的 `// removed` 注释
- 未使用的向后兼容 shim

**允许的防御性代码：**

- 系统边界处的输入校验（用户输入、外部 API 响应）
- I/O 操作的错误处理
- 可空数据库字段的 null 检查
- 测试代码中的类型断言

**判断标准：** 这个检查是否在防御一个**实际可能发生**的情况？如果值在类型系统中已经保证非空，就不需要运行时检查。

### 3. 结构卫生

**禁止的嵌套模式（超过 2 层）：**

```
❌ 深层嵌套：
if x:
    if y:
        if z:
            do_something()

✅ 早返回：
if not x: return
if not y: return
if not z: return
do_something()
```

**禁止的复杂表达式：**

```
❌ 嵌套三元：
result = a if b else (c if d else e)

✅ 显式条件：
if b:
    result = a
elif d:
    result = c
else:
    result = e
```

### 4. 冗余代码卫生

**禁止的冗余模式：**

- 重复的代码块（应提取为函数）
- 重复的测试用例（应参数化）
- 未使用的导入/变量
- 空的 catch 块 `catch(e) {}`
- 无操作的函数/方法

## 安全边界

**以下情况不适用此规则：**

1. **永远不移除 I/O、网络、文件操作的错误处理**
2. **永远不简化用户输入或外部数据的校验**
3. **永远不改变公共 API 签名**
4. **永远不移除类型注解**（即使看起来冗余）
5. **如果某个模式在多处出现，可能是有意为之**——先确认再修改

## 自检流程

编写代码时，对每一段新代码问自己：

1. **这个注释是否在解释"为什么"？** 如果只是解释"是什么"→ 删除
2. **这个检查是否在防御实际可能的情况？** 如果类型系统已保证 → 删除
3. **这个嵌套能否用早返回展平？** 如果可以 → 重构
4. **这段代码是否和其他地方重复？** 如果是 → 提取

## 与 review skill 的关系

reviewer 在审查时应检查代码卫生问题。如果发现 AI 异味模式，标记为 `minor` 级别问题并建议修改。

## 常见 AI 异味示例

| 模式 | 问题 | 修复 |
|------|------|------|
| `// 初始化变量` + `let x = 0` | 废话注释 | 删除注释 |
| `if (result !== null && result !== undefined)` | TypeScript 中可用 `result != null` | 简化 |
| `try { return obj.key } catch { return null }` | 对象字面量访问不会抛异常 | 移除 try-catch |
| `/** @param name 用户名 */ function greet(name: string)` | 参数名已自解释 | 删除文档注释 |
| 连续 3 层 if 嵌套 | 认知负荷高 | 用 guard clause 展平 |
| `// 检查用户是否存在` + `if (user)` | 代码已自解释 | 删除注释 |
