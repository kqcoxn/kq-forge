# 研究先行（Search First）

## 核心理念

> 不要重新发明轮子。动手写代码之前，先花 5 分钟搜索现有解决方案。

## 工作流程

```
┌─────────────────────────────────────┐
│ 0. 搜索渠道预检                      │
│    确认哪些搜索渠道可用              │
├─────────────────────────────────────┤
│ 1. 需求分析                          │
│    明确需要什么功能、语言/框架约束    │
├─────────────────────────────────────┤
│ 2. 并行搜索                          │
│    ┌────────┐ ┌────────┐ ┌────────┐ │
│    │ 项目内 │ │ 包管理 │ │ GitHub │ │
│    │ 已有?  │ │ 注册表 │ │ / Web  │ │
│    └────────┘ └────────┘ └────────┘ │
├─────────────────────────────────────┤
│ 3. 评估候选方案                      │
│    功能匹配度 / 维护状态 / 许可证    │
├─────────────────────────────────────┤
│ 4. 决策                              │
│    ┌──────┐  ┌──────┐  ┌──────┐    │
│    │ 采用 │  │ 包装 │  │ 自建 │    │
│    │ as-is│  │ 扩展 │  │ 定制 │    │
│    └──────┘  └──────┘  └──────┘    │
├─────────────────────────────────────┤
│ 5. 实施                              │
│    安装包 / 写薄包装层 / 写定制代码  │
└─────────────────────────────────────┘
```

## 决策矩阵

| 搜索结果 | 行动 |
|----------|------|
| 精确匹配，维护良好，MIT/Apache 许可 | **采用** — 直接安装使用 |
| 部分匹配，基础良好 | **包装** — 安装 + 写薄包装层 |
| 多个弱匹配 | **组合** — 组合 2-3 个小包 |
| 没有合适方案 | **自建** — 写定制代码，但已被研究结果启发 |

## 搜索顺序

每次动手前，按以下顺序快速检查：

1. **项目内已有？** → 用 `rg` / `grep` 搜索相关模块和测试
2. **包管理器有？** → 搜索 npm / PyPI / crates.io / Maven Central
3. **GitHub 有参考实现？** → 搜索维护良好的开源实现
4. **官方文档有推荐方案？** → 查阅框架/语言官方文档

## 评估标准

对每个候选方案打分（1-10）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 功能匹配度 | 高 | 是否满足核心需求 |
| 维护状态 | 高 | 最近提交时间、issue 响应速度 |
| 社区规模 | 中 | 下载量、star 数、使用者数 |
| 文档质量 | 中 | 是否有清晰的 API 文档和示例 |
| 许可证兼容 | 高 | 是否与项目许可证兼容 |
| 依赖数量 | 中 | 传递依赖是否过多 |
| 包体积 | 低 | 是否引入不必要的体积 |

## 反模式

| 反模式 | 正确做法 |
|--------|----------|
| 直接写代码，不搜索 | 先花 5 分钟搜索 |
| 搜索渠道不可用时谎称"没找到" | 诚实报告哪些渠道被跳过了 |
| 为一个小功能引入巨型依赖 | 评估包体积和传递依赖 |
| 过度包装，失去原库优势 | 薄包装层，保留原 API 语义 |
| 只看 star 数不看维护状态 | 综合评估，维护状态权重最高 |

## 输出格式

```markdown
## 研究报告：[功能需求]

**需求：** [一句话描述]
**约束：** [语言/框架/许可证]

### 搜索结果

| 方案 | 来源 | 匹配度 | 维护 | 许可证 | 综合评分 |
|------|------|--------|------|--------|----------|
| [方案A] | npm | 9/10 | 活跃 | MIT | 9 |
| [方案B] | GitHub | 7/10 | 一般 | Apache | 7 |

### 决策

**推荐：** [采用/包装/组合/自建] [方案名]
**理由：** [为什么选这个]

### 跳过的渠道

- [渠道名]：[不可用原因]
```
